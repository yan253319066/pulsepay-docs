# DAO 組織

## DAOとは？

```
DAO = Decentralized Autonomous Organization
    = 分散型自律組織

類比：
- 伝統会社：ボス/経営層が意思決定
- DAO：コード + トークン保有者が意思決定

特徴：
- 中央集権的なリーダーがいない
- ルールはスマートコントラクトに記述
- メンバーが共同治理
```

## DAO vs 伝統組織

| 维度 | 伝統会社 | DAO |
|------|----------|-----|
| 意思決定方式 | 経営層が決める | コミュニティ投票で決定 |
| 所有権 | 株主持股 | Token 保有者 |
| ルール実行 | 人為的に実行 | コードが自動実行 |
| 透明性 | 内部情報が不透明 | チェーン上で公開透明 |
| 参入门槛 | 採用/入社 | Token 購入 |
| 退出方式 | 退職/解雇 | Token 売却 |
| 地理的制限 | あり | なし |

## DAO の動作原理

### 1. Token 治理

```
治理トークン = 投票権

トークン持有 = 投票権獲得
投票権重み = トークン数量

例：
- 1000 AIP を所持 → 1000 票
- 10000 AIP を所持 → 10000 票
```

### 2. 提案システム

```
提案フロー：

1. 提案作成
   誰でも作成可能（時折最低トークン门槛必要）

2. 議論期間
   コミュニティで議論、討論
   通常 2-7 日

3. 投票期間
   Token 保有者が投票
   通常 3-5 日

4. 実行期間
   提案可決 → 自動実行
   提案否決 → 破棄
```

### 3. 治理パラメータ

```
一般的な治理パラメータ：

提案门槛：
- 最低トークン数量が必要
- 例：100 AIP を所持で提案可能

投票门槛：
- 参加率に達しないと無効
- 例：少なくとも5% Token 参加

可決门槛：
- 賛成票 > 反対票
- 時折特定比率超えが必要
```

## DAO のタイプ

### 1. プロトコル DAO

```
DeFi プロトコルを管理：

代表：
- MakerDAO：DAI ステーブルコイン管理
- Uniswap：DEX プロトコル管理
- Aave：貸借プロトコル管理

意思決定内容：
- パラメータ調整
- 新機能開発
- 資金使用
```

### 2. 投資 DAO

```
コミュニティ資金で投資：

代表：
- BitDAO：暗号通貨業界投資
- The LAO：初期プロジェクト投資
- Flamingo：NFT 投資

特徴：
- コミュニティ金庫投資
- 収益分配
- リスク分散
```

### 3. ソーシャル DAO

```
コミュニティ駆動のソーシャル組織：

代表：
- Friends With Benefits
- Krause House（スポーツ）

特徴：
- 会員制
- 共通兴趣
- ソーシャルキャピタル
```

### 4. メディア DAO

```
分散型コンテンツ創作：

代表：
- Bankless
- NewsDAO

特徴：
- コンテンツクリエイターコミュニティ
- トークンで創作激励
- 分散型編集
```

## DAO の優勢

### 1. 透明公正

```
优点：
- 全ての意思決定がチェーン上で公開
- ダークボックス操作不可
- ルール前は人人平等

対比：
伝統会社の意思決定は不透明
DAO は誰でも検証可能
```

### 2. グローバル参加

```
优点：
- 地理的制限なし
- 24/7 参加可能
- 誰でも参加可能

対比：
伝統会社は雇用関係あり
DAO はトークン所持のみ
```

### 3. コード即法律

```
优点：
- ルールが自動実行
- 経営層を信頼不要
- 人為的ミスを減少

例：
投票可決 → スマートコントラクトが自動実行
人的介入不要
```

### 4. 激励整合

```
优点：
- Token 保有者 = 利害関係者
- 誰でも組織をよくしたい
- 長期的な利益整合

例：
AIP 上昇 → みんなが嬉しい
プロトコル発展 → Token 価値上昇
```

## DAO の課題

### 1. 投票率低

```
問題：
- 多くのメンバーが投票しない
- 意思決定が大口に支配

解決策：
- 委任投票（Delegation）
- 自動投票戦略
- 投票参加激励
```

### 2. 治理攻撃

```
問題：
- 大口が投票を操作可能
- 51% 攻撃

解決策：
- タイムロック（遅延実行）
- マルチシグ门槛
- クールダウン期間
```

### 3. 法的曖昧

```
問題：
- DAO は法的实体？
- メンバーの責任？

現状：
- 一部国家は立法を開始
- 越来越多的 DAO が LLC 登録
- 法的枠組みが逐步的に整備
```

### 4. 実行困難

```
問題：
- チェーン上の意思決定をチェーン下で強制実行不可
- 実行者を信頼必要

解決策：
- マルチシグ钱包
- 役割分離
- 透明な実行
```
