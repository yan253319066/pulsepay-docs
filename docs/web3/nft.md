# NFT 完全指南

## 什么是 NFT？

```
NFT = Non-Fungible Token = 非同质化代币

关键概念：
- 同质化 (Fungible)：可以互换
  例：1美元 = 1美元，可以互换
- 非同质化 (Non-Fungible)：独一无二
  例：蒙娜丽莎 = 只有一幅，独一无二

NFT = 在区块链上的"数字蒙娜丽莎"
```

## NFT 与传统数字资产的区别

| 维度 | 传统数字资产 | NFT |
|------|-------------|-----|
| 所有权证明 | 平台内部记录 | 区块链公开可查 |
| 唯一性 | 可以无限复制 | 每个都独一无二 |
| 可交易性 | 受平台限制 | 可自由交易 |
| 跨平台 | 不能 | 可以跨平台转移 |
| 创作者收益 | 一次性 | 持续获得版税 |

## NFT 的技术标准

### ERC-721（标准 NFT）

```
最常见的 NFT 标准：

```solidity
interface IERC721 {
    // 转账
    function transferFrom(address from, address to, uint256 tokenId) external;
    
    // 查询所有者
    function ownerOf(uint256 tokenId) external view returns (address);
    
    // 查询 Token URI
    function tokenURI(uint256 tokenId) external view returns (string);
}
```

特点：
- 每个 Token ID 唯一
- 可以追踪每个 NFT 的所有者
- 包含元数据（图片、属性等）
```

### ERC-1155（多 token 标准）

```
更高效的标准：

特点：
- 一个合约管理多种 NFT
- 支持同质化和非同质化混合
- 节省 Gas 费

用途：
- 游戏道具
- 批量发行
- 票务系统
```

## NFT 的风险

### ⚠️ 市场风险

```
NFT 价格波动极大：
- 可能一天涨跌 50%+
- 流动性差
- 难以变现

防范：
- 只投资你能承受损失的钱
- 不要 FOMO（错失恐惧）
- 做好研究再购买
```

### ⚠️ 版权风险

```
问题：
- 买了 NFT ≠ 拥有版权
- 版权仍在创作者手中
- 不能商用

防范：
- 仔细阅读 NFT 条款
- 确认版权归属
```

### ⚠️ Rug Pull 风险

```
问题：
- 项目方卷款跑路
- NFT 价格归零

防范：
- 调查团队背景
- 查看合约是否开源
- 社区是否活跃
```

### ⚠️ 存储风险

```
问题：
- NFT 存储在链上
- 图片可能存储在链下（IPFS/中心化服务器）
- 图片可能失效

防范：
- 确认图片存储在 IPFS
- 使用 Arweave 等去中心化存储
- 保留备份
```
